ABSTRACT

The Smart Voting System (GoVote) is a highly secure, full-stack digital solution engineered to eliminate local voting fraud and streamline the election lifecycle. The main objective of this application is to digitalize democratic processes by implementing stringent dual-level security algorithms seamlessly over a user-friendly interface.

Functionality: Users interact with a dynamic portal governed by EmailJS live One-Time Password (OTP) verification. Voters are restricted to one vote per role per election mathematically. The system features a distinct Admin Control Panel shielded by secondary OTP gateways, allowing authorized personnel to launch elections, force-end polls, and instantly download printable, single-page official PDF/HTML Election Certificates containing final vote aggregations.

Technical Stack: The architecture implements a robust three-tier layout. The Frontend operates purely on React 19 (Vite), leveraging dynamic state changes. The Backend utilizes the Spring Boot framework to securely compute REST APIs, while MySQL reliably stores vote constraints mathematically to prevent database tampering.

OBJECTIVES OF THE PROJECT

  • User Interface: To design a highly responsive, modern React interface facilitating swift, intuitive mobile navigation for varying demographics of voters.
  • Functional Features: To integrate EmailJS API protocols that securely blast live 6-digit cryptographic OTP tokens automatically to remote voter emails.
  • System Logic: To engineer rigorous voter-lock parameters, guaranteeing that multiple identities mapped to similar combinations cannot artificially skew automated ballot returns.
  • Technical Integration: To construct dedicated Administrative REST API gateways via Spring Boot that control the start-times, lifecycles, and cryptographic result generations of live institutional elections.

FLOW CHART

[ Diagram Representation ]

Start → Request Secure OTP → Verification Gateway → Active Elections Dashboard → Cast Vote (Lock Logic) → Admin Panel Verification → Dashboard & Invoice Download (End)

Flow Description:

  1. Start & Login: The user attempts access by submitting their Email and tracking phone number.
  2. Validation Gateway: The server generates a random token and hooks into EmailJS to dispatch the physical payload. The user inputs the code to breach the lock.
  3. Home Dashboard: Voters evaluate active elections. Time-lapsed elections are natively excluded based on algorithmic clock comparisons.
  4. Action (Vote Integrity Lock): The user selects a candidate. The system checks the `mock_votes` algorithm. Standard verification allows the payload update; duplicate attempts hit immediate blocks.
  5. Admin Protocol: If an Admin enters, they face a secondary physical OTP lock before launching or commanding forced election endings.
  6. Confirmation (End): After closure, Admins generate single-page HTML cryptographic certificates displaying the verified vote counts.
