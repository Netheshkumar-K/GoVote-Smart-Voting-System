ABSTRACT

The Online Food Ordering System is a full-stack digital solution designed to modernize the traditional restaurant and cafeteria catalog process. The main objective of this application is to bridge the gap between consumers and food providers by offering a highly efficient, remote digital interface capable of handling hundreds of concurrent orders seamlessly.

Functionality: Users interact with a dynamic, mobile-style interface allowing them to intuitively browse categorized food menus, review pricing, update quantities in a digital cart, and securely check out. The system features immediate validation, reducing wait times and manual operational errors.

Technical Stack: The architecture strictly implements a modular three-tier layout. The Frontend operates purely on HTML5, CSS3, and JavaScript logic. The Backend is deeply powered by the robust Spring Boot library managing REST APIs, and the Database persists all user and inventory states reliably using MySQL.

OBJECTIVES OF THE PROJECT

  • User Interface: To design a simple, intuitive, and responsive mobile-style interface facilitating swift navigation across food categories without page reloads.
  • Functional Features: To integrate dynamic browsing features such as real-time filtering, keyword search tools, and immediate cart cost calculations.
  • System Logic: To engineer a rigorous order management system that tracks cart variables, intercepts duplicate order anomalies, and ensures data integrity before official checkout confirmation.
  • Technical Integration: To construct secure REST API corridors using Spring Boot controllers that seamlessly exchange massive JSON payloads between the client interface and the remote MySQL persistence layer.

FLOW CHART

[ Diagram Representation ]

Start → Login Authorization → Home Menu Dashboard → Cart Management → Order Confirmation → End

Flow Description:

  1. Start & Login: The user accesses the landing page and inputs authentication credentials.
  2. Validation: The system validates credentials against database tables. Upon success, passage is granted.
  3. Home Dashboard: The user is presented with the dynamically rendered food catalog retrieved via API.
  4. Action (Selection & Search): The user utilizes the search functionality to pinpoint specific foods and adjusts cart quantities.
  5. Cart Evaluation: The system independently calculates the accumulated pricing total and prepares the receipt array.
  6. Confirmation (End): Final checkout occurs, storing the official order in the MySQL database and logging out the user safely.

PROGRAM

Backend Source Code (Spring Boot):

  • Entities (Food.java): Defining the `@Entity` parameters mapped precisely to relational tables in the remote database.
  • Repositories (FoodRepository.java): Extending `JpaRepository` to abstract heavy SQL scripting into clean method calls.
  • Controllers (FoodController.java): Establishing `@RestController` endpoints mapping network traffic to the core business logic.
  • Configuration (application.properties): Managing specific MySQL URLs, driver class names, and active port commands.

Frontend Source Code:

  • HTML Structure (index.html): Creating semantic form elements, login wrappers, and responsive table skeletons.
  • CSS Styling (style.css): Utilizing modernized flex layouts, interactive hover states, and smooth color palettes.
  • JavaScript Logic (script.js): Injecting fetch algorithms, asynchronous DOM manipulation, and frontend mathematical validations.

Database Scripts (MySQL):

  • SQL Scripts: Generating necessary user tables via `CREATE TABLE` routines and seeding preliminary food catalog states using explicit `INSERT` operations.
